thfAdaptTaskCreate

Syntax

#include <trulyhandsfree.h>
int thfAdaptTaskCreate(thf_t **thf, adaptObj_t *aObj, char *taskTypeName, 
                       char *taskDataFileName, char *taskName);

Description

Create an instance of a task, adding it to an 'adapt' object. The task-specific data are read in from the specified filename. The task name will identify this task in all subsequent calls.

Parameters

Type Name Dir. Description
thf_t * *thf in Session object.
adaptObj_t * aObj in adapt object.
char * taskTypeName in description of the type of task being created. Currently the only valid name is 'trigger'
char * taskDataFileName in filename that contains task-specific data.
char * taskName in a user-defined name for the task being created.

Return

Returns 1 on success or 0 on error.

Errors

String Description
adapt object not found A NULL pointer was passed in for the adapt object; pass in a correctly-initialized adapt object pointer created by thfAdaptCreate.
adapt object not initialized A non-NULL pointer was passed in for the adapt object, but this pointer does not contain a properly-initialized adapt object that is the output of thfAdaptCreate.
tasktype name must be non-NULL A NULL pointer was passed in for the task type name. The only valid name is currently a character string containing 'trigger'
task data file must be non-NULL A NULL pointer was passed in for the task datafile. A valid filename must be used, and this filename must contain task data in a valid format.
task name must be non-NULL A NULL pointer was passed in for the task name. The task name can be any (arbitrary) string of one or more ASCII characters with no spaces, but must have non-zero length.
error reading task data file The task data file is not valid for the specified task type.
unknown task type; must be 'trigger' The only supported task type is currently 'trigger'
task data file is not for the type 'trigger' Although the task data file is valid, it is for a different type of task than the specified task type.
error destroying task data Internal error destroying task data. Contact customer support.
