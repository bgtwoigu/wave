deepdump

Syntax

TrulyHandsfree SDK\<platform>\bin\deepdump [options] -n netfile -s searchfile -t target -T trigger -x outputnetfile -y outputsearchfile

Description

Given compiled network and search files and a DSP target string, output files in deeply-embedded binary format. Contact Sensory for a list of supported DSP targets. Options include: -C for specifying a configuration file. A header file is always generated, as of SDK v3.12.0.

Parameters

Name Default Description
-C file Config file to write non-default parameter values to the dumped output.
-n netfile Neural network (acoustic model) file name.
-s searchfile Compiled phrase-spotting search file name.
-t target A string identifying the DSP target architecture (e.g., "pc36")
-T trigger use 1 if the search is continuously listening, or 0 if it is windowed.
-x Output file name used for dumped acoustic model.
-y Output file name used for dumped search.

Config File Format

The format of the config file is the parameter, followed by a space, followed by the value. The following parameters may be adjusted:

DSP_SV_THRESH, which is configured using thfSpeakerConfigSet with the SPEAKER_DSP_SV_THRESH key. This parameter is described in thfSpeakerConfig_t.

PARAMA_OFFSET, which is configured using thfPhrasespotConfigSet with the PS_PARAMA_OFFSET key. This parameter is described in thfPhrasespotConfig_t.

EPQ_ENABLE and EPQ_MIN, which are configured using thfRecogConfigSet with the REC_EPQ_ENABLE and REC_EPQ_MIN keys. These parameters are described in thfRecogConfig_t.

Below is an example config file.

DSP_SV_THRESH 0.35
PARAMA_OFFSET 100
EPQ_ENABLE 1
EPQ_MIN -8.0

If EPQ_ENABLE is not specified but EPQ_MIN is specified, then EPQ will be enabled.
