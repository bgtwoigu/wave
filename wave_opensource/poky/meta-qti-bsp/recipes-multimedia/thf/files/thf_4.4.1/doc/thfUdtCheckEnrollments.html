thfUdtCheckEnrollments

Syntax

#include <trulyhandsfree.h>
int thfUdtCheckEnrollments(thf_t **thf, udt_t *udt, thfUser_t *user, 
                           unsigned short whichUser, 
                           unsigned long *globalFeedback, 
                           unsigned long *feedbackArray, float *phraseQuality);

Description

Using the udt object and user data structure, provide feedback on the quality of the phrase and on the quality of the enrollment recordings for a specific user.

The phrase quality is a measure of how well the trigger phrase is likely to perform, in terms of false rejects and false accepts. The quality of the enrollment recordings is provided in two variables: (1) a 'global' feedback that indicates, in a single variable, if there are problems detected in any of the enrollment recordings (and if so, what kinds of problems are detected), and (2) an enrollment-specific feedback array that indicates, for each enrollment recording, if any problem is detected and what the detected problems are. A value of NULL can be passed in for phrase quality, global feedback, and/or enrollment-specific feedback, if no output of that type is desired. The global feedback and enrollment-specific feedback provide the same kind of feedback, but at different levels of detail.

Quality measure thresholds have reasonable default values, but can be modified using thfUdtConfigSet.

Parameters

Type Name Dir. Description
thf_t * *thf in Session object.
udt_t * udt in Udt object.
thfUser_t * user in array of user data structures, containing all enrollment recordings for all users.
unsigned short whichUser in index into the array of user data structures, indicating which user to provide feedback for. This value must be at least zero, and less than the length of the array 'user'.
unsigned long * globalFeedback out NULL, or a single value indicating problems with any of the enrollment recordings. If NULL, no global feedback is provided. If not NULL, then if this value is 0, no problems were detected in any of the enrollment recordings. If this value is not 0, then the specific problem(s) can be determined from the bit fields, as specified in the list of recording quality parameters.
unsigned long * feedbackArray out NULL, or array indicating problems per enrollment recording. If not NULL, feedbackArray must be allocated before calling thfUdtCheckEnrollments(). For a particular enrollment index 'eIdx' (from 0 through the number of enrollments minus 1), the value of feedbackArray[eIdx] is 0 if no problems were detected for this enrollment. A non-zero value indicates one or more detected problems. The specific problem(s) can be identified from the bit fields, as specified in the list of recording quality parameters. If feedback on the individual enrollment recordings is desired, a one-dimensional array of 'unsigned long' must be allocated and passed in as the 'feedbackArray' parameter.
float * phraseQuality out NULL, or a value from 0.0 to 1.0 indicating the estimated quality of the enrollment phrase. A value of 0 indicates extremely poor quality, and a value of 1.0 indicates very good quality. Very low values indicate that the user should re-enroll until a better value is obtained; we recommend a cutoff of at least 0.35, below which the user must re-enroll. More detail is provided in the enrollment-checking section of how to enroll a user-defined trigger.

Return

Returns 1 on success or 0 on error.
