PhraseSpotting Grammars:

NOTE: Phrasespotting is orthogonal to and preferred over the older word-spotting technology.

Phrasespotting is implemented using Sensory's TrulyHandsfree™ Voice Control 3.0 technology. Phrasespotting is commonly used to implement trigger phrase(s) or for continuous-listening style applications.

Refer to the PhraseSpot Sample provided with the SDK distribution.

A phrasespot vocabulary can either be loaded from file (see thfSearchCreateFromFile) or specified from scratch as a grammar (see thfSearchCreateFromGrammar). NOTE: phrasespotting is not supported with Lists. Phrasespotting is initiated by passing the PHRASESPOTTING argument to thfSearchCreateFromGrammar. There are also additional parameters that can be configured using thfPhrasespotConfigSet.

A phrasespotting grammar is a simple collection of phrases to be spotted plus a set of parameters for each phrase. A grammar with a single phrasespot phrase might look as follows: 

      grammar = this is a trigger;
    paramA:$grammar -1000;
    paramB:$grammar 610;
We specify 'paramA' and 'paramB' (and optionally 'paramC') for each phrase in the grammar. These parameters alter the recognition behavior. The parameters must be tuned for each individual grammar. Unfortunately there is no one-size-fits-all. Suitable parameter values (i.e., paramA, paramB, paramC) should be determined empirically using both in-vocabulary and out-of-vocabulary recordings. See Phrasespot Parameters for further details. Contact Sensory for assistance with tuning phrasespot parameters.

Notice that the grammar does not include *nota or employ the use of surrounding silence *sil unlike a conventional grammar. This is because we want to be able to spot the phrase in continuous speech, rather than require the user to pause before and after the phrase. Phrasespotting usage does not require a speech detector, however, it could be paired with one to force the recognition to time-out after a specified period.

Multiple phrases can be supported too, for example: 

      g1 = this is a phrase;
  g2 = this is another phrase;
  g = $g1 | $g2;
  paramA:$g1 -1000;
  paramB:$g1 610;
  paramA:$g2 -750;
  paramB:$g2 500;
Here each phrase is assigned to a separate variable (i.e., g1, g2). The complete grammar 'g' is then specified in term of these variables. Finally, parameters 'paramA' and 'parmaB' are defined for each phrase/variable. Again the parameters must be tuned specifically for the grammar. Tuning is more involved for multiple phrases because the number of possible permutations are much larger.
