Defining a Vocabulary Using a Phrase List (Easy)

A phrase list specification is simply an enumerated list of all the phrases that make up the vocabulary. The following steps are involved in creating a search object from a phrase list specification:
  1. Define a list of words or phrases to be recognized.
  2. Optionally define pronunciations for each phrase (see How to generate pronunciations). NOTE: The order of the pronunciation list must correspond with the order of the phrase list. If no pronunciations are provided, they will be generated internally. For optimal recognition performance, it's recommended you provide them explicitly so you can verify accuracy and possibly even tune them for your task. A tool, CreatePronun, is provided that, given a phrase list, generates an ordered list of pronunciations, which can then be edited before inclusion in your code.
  3. Using the phrase list and pronunciations (optional), compile the vocabulary specification to create a search object by calling thfSearchCreateFromList. This search object contains a binary representation of the vocabulary.
  4. Utilize the search object for recognition by calling thfRecogInit and/or save it to file for later use by calling thfSearchSaveToFile.
  5. Destroy the search object when no longer needed by calling thfSearchDestroy.

Example Phrase List Specification

const char *phrases[] = {
  "One",
  "Two",
  "Three",
  "This is a test",
  "Hello world",
  "*nota"
};
