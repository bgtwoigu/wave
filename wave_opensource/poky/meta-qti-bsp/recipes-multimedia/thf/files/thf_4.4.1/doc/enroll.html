Sensory Confidential

enroll

Syntax

TrulyHandsfree SDK\<platform>\bin\enroll [options] enrollfiles...

Description

A tool for performing User-Defined Trigger (UDT) and/or Enrolled Fixed Trigger (EFT) enrollment: Creates compiled search and acoustic model files from example recordings for each phrase/user. The resulting models can be evaluated using the pseval tool. The console output includes recommended settings for several pseval optional parameters.

Parameters

Name Default Description
-t file Task data filename, specific to the development of a UDT trigger in a particular language or a specific phrase for an enrolled fixed trigger. This source file is required.
enrollfiles... Enrollfiles, supplied in groups of wavefiles separated by the user name (name of user being enrolled) preceded by a '+'. The user name will be the result that the recognizer provides when that user's phrase is spotted. The wavefiles can be in RIFF (.wav) format with 16-bit linear PCM encoding, or a headerless 16-bit linear PCM file in host byte order. If the latter, the -r rate flag is required to specify the sample rate. These parameters are to be omitted if enrollment data is read from a cache with the -x option.
-f disabled Disable audio check and force evaluation.
-n file net.raw Neural network (acoustic model) file name. This is one of the output files created.
-s file search.raw Compiled phrase-spotting search file name. This is one of the output files created.
-r rate read from wavfile Audio sample rate in Hz, required if the wave file is in PCM format.
-x file A cache file that contains all information about users and enrollments. If this file does not exist, it is created after enrollment. If this file does exist, it is read. If this file is read, then no user or enrollment-file information should be specified on the command line. in PCM format.
-v disabled Produce verbose progress messages.
-V disabled Produce very verbose messages (more information than is generated with only the -v option).

Examples

enrollfiles are supplied in groups of wavefiles, separated by the speaker name preceded by a '+'. For example:
+username1 user1_enr_1.wav user1_enr_2.wav +username2 user2_enr_1.wav user2_enr_2.wav

For audio files in PCM format, all recordings must be sampled at the same rate, which must be given with the -r option. This sampling frequency must be equal to or greater than the sampling frequency associated with the task data file.
