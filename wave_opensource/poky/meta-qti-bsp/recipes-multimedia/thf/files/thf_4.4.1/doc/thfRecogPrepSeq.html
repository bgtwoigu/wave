thfRecogPrepSeq

Syntax

#include <trulyhandsfree.h>
int thfRecogPrepSeq(thf_t **thf, recog_t *dst, recog_t *src);

Description

Prepares a recognizer for sequential operation by copying buffered features from a source phrase spotter. This is useful, for example, when a trigger phrase is followed by a command without intervening silence ("Hello blue genie volume up.")

The source (phrase spot) and destination recognizers must use compatible feature sets.

Prepare the source recognizer by calling thfPhrasespotConfigSet with the PS_SEQ_BUFFER key and a value specifying the size of the buffer. The buffer must be at least the size (in ms) of the largest audio block passed to thfRecogPipe, or features will be lost and thfRecogPrepSeq will report an error.

Recommended use: When thfRecogPipe(thf, src, ...) returns a stateResult of RECOG_DONE, call thfRecogPrepSeq(thf, dst, src) and then pass subsequent audio blocks to the destination recognizer with thfRecogPipe(thf, dst, ...)

Parameters

Type Name Dir. Description
thf_t * *thf in Session object.
recog_t * dst out Destination recognizer object.
recog_t * src in Source recognizer object.

Return

Returns 1 on success or 0 on error.

Errors

String Description
src recognizer not configured for sequential recognition (set PS_SEQ_BUFFER > 0) The sequential recognition capability was not enabled in the source recognizer.
data for the last spot in src are not available in the buffer (increase PS_SEQ_BUFFER size?) Sequential recognition is enabled in the source recognizer, but this recognizer has seen more audio since the last phrase spot than the configured buffer can hold.
src and dst recognizers use incompatible features The source and destination recognizers use recognition feature sets that are not compatible. These two recognizers cannot be used sequentially.
failed to copy feature buffers The feature buffers could not be copied from the source recognizer.
failed to copy audio buffers The audio buffers could not be copied from the source recognizer.
