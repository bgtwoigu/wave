SDK Components

The SDK is available for various platforms (e.g., ix86-win, ix86-linux). Please make sure to refer to the accompanying TrulyHandsfree SDK\<platform>\Readme.txt file for platform-specific information.

The SDK distribution includes the following directory structure relative to the <install> directory: 
+ TrulyHandsfree SDK: contains entire SDK distribution.
  + Data: contains SDK language packs
    + <language> (e.g., en_us_16kHz_v9): contains language-specific
      acoustic models, pronunciation models and documentation.

  + include: contains headers with API definitions.
  + <platform> (e.g., ix86-win): contains platform-specific
    binaries and documentation.
    + bin: contains tools and executable samples.
    + lib: contains static libraries.
    + Samples: contains sample code.
      + Data: contains data files used by the samples.
      + Shared: contains support code used by the samples.
We now describe each of these directories and their content.

Data Files "TrulyHandsfree SDK\Data\<language>":

All language-specific data is contained in a single folder; referred to as a language pack. Two types of data files are provided in the data directory: Acoustic Models and Pronunciation Models.

The acoustic models are implemented as neural networks, often referred to as nets. A selection of different size nets are provided using the following naming conventions: 

  nn_<language>_<dialect>_<samplerate>_<framesize>_<hidden layer size>_<version>.raw

The pronunciation model files use the following naming convention: 

  lts_<language>_<dialect>_<version>.raw

The folder also contains a Help file that describes the acoustic model characteristics, including the list of phonemes supported and relative performance of the different size acoustic models.

Include Files "TrulyHandsfree SDK\include":

The SDK header file trulyhandsfree.h contains a complete list of API definitions and exported functions. Include this header file in your application source code to access SDK API functions.

The header sensorytypes.h is included by code generated by the DumpRaw utility for use with static data.

Library Files "TrulyHandsfree SDK\<platform>\lib":

The SDK is delivered as a static library. Several build variants are possible, each of which contains a different subset of API functions so you can select the one appropriate for your needs, as follows:

  • thf: contains ALL available API functions (for both building and running a recognizer); Requires a file system.
  • thfNoFS: same as thf, minus functions that involve reading or writing to file.
  • thfRecog: contains API functions for running a recognizer only; Requires a file system.
  • thfRecogNoFS: same as thfRecog, minus functions that involve reading or writing to file.

NOTE: The current SDK only ships with the thf library. Please contact Sensory if you are interested in one of the other build variants.

NOTE: Make sure you use the library for the appropriate platform.

NOTE: Library filenames may vary slightly depending on platform. For instance, ix86-linux library filenames include the prefix "lib" (i.e., "thf" becomes "libthf.a").

NOTE: On some platforms (Android, for example) CPU-specific variants are included, for example libthf_armeabi-v7a.a or libthf_mips.a.
