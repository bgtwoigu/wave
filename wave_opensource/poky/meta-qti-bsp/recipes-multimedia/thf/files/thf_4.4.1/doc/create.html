Creating a Recognizer

In this section, we describe the following steps:
  1. Create a recog object
  2. Create a search object
  3. Configure the recognizer
  4. Initialize the recognizer

Step 1: Create a Recog Object

The recog object is the main data structure of the recognizer. Create it with thfRecogCreateFromFile that will return a unique handle to the object. It requires you to specify an acoustic model file to load, the maximum audio buffer size, the maximum backoff size and whether or not you want to use a speech detector.

A selection of speaker-independent acoustic models is provided in the SDK's TrulyHandsfree SDK\Data directory. These vary in size and purpose, including general purpose and task-specific acoustic models. In general, larger acoustic models are more accurate but require more computation. Some are trained on 8kHz and others at 16kHz. Choose the one that is most appropriate for your task and hardware.

It is feasible to create and maintain multiple recog objects in memory, if needed. Each object has a unique handle. In one application one might have the need for a task-specific acoustic model such as digit strings and a general purpose acoustic model for the generic vocabularies.

Step 2: Create a Search Object

The search object specifies the recognition vocabulary and was explained in How to define the recognition task section. Additionally, you can load a pre-compiled search object from file using thfSearchCreateFromFile.

It is feasible to create and maintain multiple search objects in memory, if needed. Each object has a unique handle. In a typical dialogue one can switch between vocabularies by swapping in the appropriate search object when needed.

Step 3: Configure the Recognizer

Many parameters influence recognition behavior, such as the size, speed and accuracy of the recognizer. The recog and search objects are created with default values that should be configured as needed. Use thfRecogConfigSet and thfSearchConfigSet to get and set recog and search object parameters, respectively. See Parameters for a description of available parameters and their defaults values.

In general, parameters can be configured at any time. However, some parameter changes will require reallocation of buffers and on platforms where memory reallocation can only occur at predictable times, make sure to configure parameters prior to initializing the recognizer.

Step 4: Initialize the Recognizer

The recognizer must be initialized with thfRecogInit before it can be used. This associates the search object with the recognizer, amongst other things.

Also, you can use thfRecogInit to change the vocabulary of an existing recognizer, simply by calling thfRecogInit with the new search object. All internal data structures (i.e., recognizer, search, and speech detector) are reset during this call.

NOTE: The search object is acoustic model specific. Ensure the recog object uses the same acoustic model file for building the search and recognizing with the search.
