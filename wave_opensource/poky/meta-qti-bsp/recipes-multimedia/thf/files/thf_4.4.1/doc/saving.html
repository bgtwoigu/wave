Saving a Recognizer and Search

The parameters used by a recognizer and/or search may be modified by calls to thfRecogConfigSet, thfPhrasespotConfigSet, or thfSearchConfigSet.

When saving a recognizer and/or search to file using thfSpeakerSaveRecogToFile, thfSearchSaveToFile, or thfUdtSaveToFile, then the following parameters are written to file and will be used when reading those files, until another call to thfRecogConfigSet, thfPhrasespotConfigSet, or thfSearchConfigSet. is made for that parameter.

The following recognizer parameters are saved in the recognizer file:

  • epqMin
  • epqEnable
The following phrasespotting parameters are saved in the search file:
  • Longpen
  • Delay
The following search parameters are saved in the recognizer file:
  • Garbage
  • Any
  • Nota

Other configurable parameters, e.g. parameters related to speech detection (Lsilence, Tsilence, etc) and enrollment-checking parameters (e.g. poorRecordings, clippingPercent, etc.) are not saved. For phrasespotting, PARAMA_OFFSET, beam, and absbeam are not saved. For the search, pruning and beam are not saved.

The motivation behind which parameters are saved and which are not is that EPQ, longpen, delay, garbage, any, and nota are things that the developer will want to set once for a particular recognizer/search. The developer will then not have to worry about setting the same values in every application. On the other hand, speech-detection and enrollment-checking parameters, paramAoffset, beam, and absbeam will need to be decided upon differently for each application.

One exception to this is if the developer does speaker verification and phrasespotting using the Recommended API, and sets a value for PARAMA_OFFSET (paramAoffset) using thfPhrasespotConfigSet before adaptation. In this case, the value of paramAoffset is related to the adaptation parameter values (different adaptation parameter values may require a different paramAoffset), and so this offset is added to the internal paramA value when the search is written to file. Therefore, the developer doesn't have to know at run-time what paramAoffset was used for adaptation.
