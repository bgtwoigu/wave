Sensory Confidential

Data Structures

The following data structures are used to maintain the state of the recognition engine. These are internal data structures whose members are hidden from the user. You will be provided with a unique handle to these structures when appropriate and can manipulate their contents via API calls.

Session object (thf_t)

The session object contains high-level SDK-specific data. Most SDK functions require this object as a parameter. You should begin your SDK utilization by creating this object and end your SDK utilization by destroying this object.
  • Create with: thfSessionCreate.
  • Destroy with: thfSessionDestroy

Recognizer object (recog_t)

  • Create with: thfRecogCreateFromFile, thfRecogCreateFromStatic.
  • Destroy with: thfRecogDestroy

Search object (searchs_t)

  • Create with: thfSearchCreateFromFile, thfSearchCreateFromStatic, thfSearchCreateFromGrammar, thfSearchCreateFromList.
  • Destroy with: thfSearchDestroy

Pronunciation object (pronuns_t)

  • Create with: thfPronunCreateFromFile, thfPronunCreateFromStatic.
  • Destroy with: thfPronunDestroy

Text-to-Speech object (tts_t)

  • Create with: thfTtsCreateFromFile, thfTtsCreateFromStatic.
  • Destroy with: thfTtsDestroy

Static search model (ifsearchC_t)

The data structure that encapsulates the read-only component of a search object in application memory.
  • Create with: DumpRaw, from a previously compiled search that was saved using thfSearchSaveToFile, and compile into application.

Static acoustic model (recogC_t)

The data structure that encapsulates the read-only component of a recognizer object in application memory.
  • Create with: DumpRaw, from supplied SDK acoustic model, and compile into application.

Static pronunciation model (pronunC_t)

The data structure that encapsulates the read-only component of a pronunciation object in application memory.
  • Create with: DumpRaw, from supplied SDK pronunciation model files, and compile into application.

Static data object (sdata_t)

The data structure that stores the number of static objects of a specific type (e.g. recogC_t, ifsearchC_t or pronunC_t) and a pointer to each. Defined in trulyhandsfree.h.

Adaptation object (adaptObj_t)

The data structure containing information about zero or more adaptation tasks. This structure will include everything needed by the adaptation functions, including user information, enrollment data, and generated models.
  • Create with: thfAdaptCreate
  • Destroy with: thfAdaptDestroy

Adaptation task information object (thfAdaptTaskInfo_t)

The data structure containing information about the task (read in from a task data file).
  • Create with: thfAdaptTaskInfo
  • Destroy with: thfAdaptTaskInfoDestroy
Most of the parameters in the data structure can be read with thfAdaptTaskGet using the specified "parameter name" to identify the parameter. If a parameter can not be read with this function, the 'Parameter Name' field is marked N/A. Some of the parameters (as indicated below) can be set with thfAdaptTaskSet using the specified "parameter name" character string to identify the parameter. The data structure contains the following:

Type Variable Name Parameter Name Description
char * taskID N/A This field specifies an identifier for the task. This identifier provides a general indication of what the trigger is, for example 'UDT' or 'hello blue genie'. This parameter can not be set, nor can it be queried with thfAdaptTaskGet. It can only be obtained with a call to thfAdaptTaskInfo.
char * taskLanguage N/A THis field specifies the target language for the task. The language may be indicated as a four-letter code, for example 'US English' may be represented as 'enUS'. This parameter can not be set, nor can it be queried with thfAdaptTaskGet. It can only be obtained with a call to thfAdaptTaskInfo.
char * taskVersion N/A This field specifies the version of the task. This string can be used to confirm that the version information in the task data file is the expected version, if more than one version is possible. This parameter can not be set, nor can it be queried with thfAdaptTaskGet. It can only be obtained with a call to thfAdaptTaskInfo.
int userDefinedTrigger ADAPT_UDT This field is 1 if the task is for a user-defined trigger or user-defined passphrase; otherwise, 0. This parameter can be read but not set.
int featureType ADAPT_FEATURE_TYPE This field identifies the type of features used in processing recordings. Currently, the only valid value is '1'. This parameter can be read but not set.
int sampleRate ADAPT_SAMPLE_RATE This field identifies the sampling frequency of the recognizer. This parameter can be read but not set.
int frameSize ADAPT_FRAME_SIZE This field identifies the frame size of the recognzier. This parameter can be read but not set.
int saveEnrollmentAudio ADAPT_SAVE_ENROLLMENT_AUDIO This field specified whether or not the enrollment audio should be saved in the thfAdaptEnrollInfo_t structure. A value of 0 specifies that the original enrollment audio will not be stored anywhere in the Fluent codebase after returning from a call to thfAdaptEnrollmentAddTrigger. A value of 1 specifies that the enrollment audio will be kept in memory and copied into the thfAdaptEnrollInfo_t structure. This parameter can be both read and set. When this parameter is set, it will affect all subsequent calls to thfAdaptEnrollmentAddTrigger.
int numEnrollments ADAPT_NUM_ENROLLMENTS This field identifies the number of enrollments recommended for this task. The application may use fewer or more enrollments than the recommended number, but performance may then be different from expected results. In particular, with more enrollments the False Reject rate tends to decrease, while the Imposter Accept rate and Noise Accept rate tend to increase. This parameter can be read but not set.
int numEnrollmentsWithContext ADAPT_NUM_ENROLLMENTS_WITH_CONTEXT This field identifies the number of enrollments with context that are recommended for this task. An "enrollment with context" is the trigger phrase spoken with an immediately-following context phrase. The application may use fewer or more enrollments with contex than the recommended number. If using a different number than the recommended value, the impact on performance will probably be less than the impact of using a different value for ADAPT_NUM_ENROLLMENTS. This parameter can be read but not set.
int delay1 ADAPT_DELAY1 The delay when a trigger resulting from adaptation with this task is used with a single user.
int delayM ADAPT_DELAYM The delay when a trigger resulting from adaptation with this task is used with multiple users. (Maximum performance usually requires a longer delay for multiple users than a single user.)
float epqMin ADAPT_EPQ_MIN The threshold used in EPQ. See the epqMin parameter.
float svThresh ADAPT_SVTHRESH The default speaker-verification threshold.
int paramAoffset ADAPT_PARAM_A_OFFSET The default value for the PARAMA_OFFSET used during phrasespotting. See Constants for more information about PS_PARAMA_OFFSET.
int doSpeechDetect ADAPT_DO_SPEECH_DETECT This field specifies whether speech detection should be performed on the enrollment recordings during adaptation. This additional step is usually not necessary, as speech detection will have been used to collect the enrollment recordings.
int contextPostProcess ADAPT_CONTEXT_POSTPROCESS This field specifies whether post processing of UDT enrollments should be performed when one or more enrollments have context. A value of 1 indicates that UDT post processing will be performed; 0 indicates it won't be. Post processing can improve overall performance, in particular the enrollment quality checks, but at the cost of additional time during thfAdaptEnrollmentCheck. This parameter has no effect on EFT performance or execution time.
float checkPhraseQuality ADAPT_CHECK_TRIGGER_PHRASE_QUALITY The threshold (minimum value) for phrase quality.
float checkEnergyMin ADAPT_CHECK_TRIGGER_ENERGY_MIN The threshold (minimum value) for minium energy of the word in the recording.
float checkEnergyStdDev ADAPT_CHECK_TRIGGER_ENERGY_STD_DEV The threshold (minimum value) for standard deviation of energy within the word in the recording.
float checkSilBegMsec ADAPT_CHECK_TRIGGER_SIL_BEG_MSEC The threshold (minimum value) for silence that must preceed the beginning of the word.
float checkSilEndMsec ADAPT_CHECK_TRIGGER_SIL_END_MSEC The threshold (minimum value) for silence that must occur after the end of the word.
float checkSNR ADAPT_CHECK_TRIGGER_SNR The threshold (minimum value) for the signal-to-noise ratio of the word in the recording.
float checkRecordingVariance ADAPT_CHECK_TRIGGER_RECORDING_VARIANCE The threshold (maximum value) for the amount of tolerated variance between recordings.
float checkPoorRecordingsLimit ADAPT_CHECK_TRIGGER_POOR_RECORDINGS_LIMIT The threshold (maximum value) for the quality of an individual recording.
float checkClippingFrames ADAPT_CHECK_TRIGGER_CLIPPING_FRAMES The threshold (maximum value) for the number of frames in the recording that can contain clipping or saturation.
float checkVowelDur ADAPT_CHECK_TRIGGER_VOWEL_DUR The threshold (minimum value) for vowel duration within the word, in milliseconds.
float checkRepetition ADAPT_CHECK_TRIGGER_REPETITION The threshold (maximum value) for repetitive sounds within an enrollment.
float checkSilInPhrase ADAPT_CHECK_TRIGGER_SIL_IN_PHRASE The threshold (maximum value) for silence within a phrase.
float checkSpot ADAPT_CHECK_TRIGGER_SPOT The threshold (minimum value) for whether this recording would have spotted using a speaker-independent trigger.

Adaptation user information object (thfAdaptUserInfo_t)

The data structure for information about each user (and the enrollments for each user) currently stored for a given task.
  • Create with: thfAdaptUserInfo
  • Destroy with: thfAdaptUserInfoDestroy
The thfAdaptUserInfo_t data structure contains the following:

Type Name Description
int numUsers The number of users known by the adapt object for this task.
thfAdaptUser_t * user A pointer to more specific information about each user.

The thfAdaptUser_t structure contains the following:

Type Name Description
char * name The username associated with this user.
int numEnroll The number of enrollment recordings that have been saved for this user
thfAdaptEnrollInfo_t * enrollInfo A pointer to more specific information about each enrollment.

The thfAdaptEnrollInfo_t structure contains the following:

Type Name Description
long ID A unique ID (unique to the current adapt object) associated with this enrollment. This ID is created by thfAdaptEnrollmentAddTrigger.
int used A binary indicator of whether or not this enrollment will be used in adaptation.
int score A user-specified score associated with this enrollment.
time_t time A user-specified time associated with this enrollment.
long phraseBeginTime The estimated time (from beginning of the enrollment audio) at which the enrollment phrase begins (in milliseconds). This estimate is approximate and tends to be somewhat "conservative"; in other words, the estimated word begin time may be a little bit after the actual word begin time. For UDT, this estimate is available immediately after calling thfAdaptEnrollmentAddTrigger, but an updated estimate is available after calling thfAdaptEnrollmentCheck.
long phraseEndTime The estimated time (from beginning of the enrollment audio) at which the enrollment phrase ends (in milliseconds). This estimate is approximate and tends to be somewhat "conservative"; in other words, the estimated word end time may be a little bit before the actual word end time. For UDT, this estimate is available immediately after calling thfAdaptEnrollmentAddTrigger, but an updated estimate is available after calling thfAdaptEnrollmentCheck.
long audioLen If the enrollment audio is saved (as specified with a call to thfAdaptTaskSet using the ADAPT_SAVE_ENROLLMENT_AUDIO parameter), then 'audioLen' specifies the length of the enrollment audio, in samples. If the enrollment audio is not saved, then this value will be 0.
short * audio If the enrollment audio is saved (as specified with a call to thfAdaptTaskSet using the ADAPT_SAVE_ENROLLMENT_AUDIO parameter), then 'audio' is a pointer to a copy of the enrollment audio samples. If the enrollment audio is not saved, then this value will be NULL.

Adaptation trigger enrollment object (thfTriggerEnrollment_t)

The data structure used when adding a trigger recording to the set of enrollments for a user, in UDT or EFT model creation. The user is responsible for allocating, populating, and freeing the data for this structure. Any unused values (e.g. reserved1) must be set to NULL or 0. The data structure contains the following:

Type Name Description
long sampFreq The sampling frequency of the audio being used to create an enrollment. (If this value is greater than the sampling frequency of the task, the audio will be converted to the appropriate sampling frequency for the task. If this value is less than the sampling frequency of the task, an error will be generated.)
short * audio A pointer to the audio of the enrollment recording, in PCM format.
unsigned long audioLen The number of samples of audio in the enrollment recordings.
long score A user-defined score associated with this enrollment. This score is currently not used.
time_t time A user-defined time associated with this enrollment. This time value is currently not used.
long enrollmentID An ID that uniquely specifies this enrollment (unique to the current adapt object). This value is set by thfAdaptEnrollmentAddTrigger and can be read after a call to this function.
char * reserved1 A reserved pointer. This value must be set to NULL.
char * reserved2 A reserved pointer. This value must be set to NULL.
char * transcription A reserved pointer. This value must be set to NULL.
char * context This parameter is NULL if the enrollment phrase is uttered in isolation (e.g. with no subsequent context phrase). If the enrollment is a trigger with a subsequent context phrase (e.g. "Hello Blue Genie, how fast is a parsec?"), then this parameter must point to a valid non-NULL character string. Any context phrase must follow the trigger; the context phrase must never precede the trigger. It is not required that this character string be the transcription of the context, but it can be.

Adaptation enrollment check object (thfEnrollmentCheck_t)

The data structure that contains results from checking one or more enrollments.
  • Create with: thfAdaptEnrollmentCheck or thfAdaptEnrollmentQuickCheck
  • Destroy with: thfAdaptEnrollmentCheckDestroy
The data structure contains the following:

Type Name Description
int allPass a binary indicator of whether or not all enrollment recordings passed all quality checks. The value is 1 if all checks were passed; else 0.
float quality The estimated phrase quality, on a scale from 0 to 1. This field is set to the phrase quality threshold for "quick" enrollment checks.
float qualityThresh The default threshold (minimum value) for phrase quality.
int numEnroll The number of enrollments that were checked. This value will be 1 for "quick" enrollment checks.
thfEnrollmentCheckOne_t * enroll A pointer to more specific information about each checked enrollment.

The thfEnrollmentCheckOne_t structure contains the following:

Type Name Description
int numChecks The number of enrollment checks. This value defines the size of each of the following arrays. The list of parameter names is defined in thfTriggerEnrollmentCheckTypes_t.
int * pass An array indicating whether or not the enrollment passed each check specified by thfTriggerEnrollmentCheckTypes_t.
float * score An array containing the score for each check specified by thfTriggerEnrollmentCheckTypes_t.
float * thresh An array containing the thresholds for each check specified by thfTriggerEnrollmentCheckTypes_t.

Adaptation trigger result (thfModelTrigger_t)

The data structure containing the output from trigger adaptation.
  • Create with: thfAdaptModelGetTrigger
  • Destroy with: thfAdaptModelDestroyTrigger
The data structure contains the following:

Type Name Description
char * taskName The name of the current task.
int numUsers The number of users who have been adapted in the last call to thfAdaptModelAdapt for this task.
float * spotParams An array of recognition parameters. The list of parameters is defined in thfTriggerEnrollParams. The recognizer (below) has been configured with these values.
int numSpotParams The number of recognition parameters; this is the size of the array 'spotParams'.
recog_t * recog A task- and user-specific recognizer returned from thfAdaptModelGetTrigger. This recognizer has already been initialized and configured with default parameters.
searchs_t * search A task- and user-specific search returned from thfAdaptModelGetTrigger. This search has already been initialized and configured with default parameters.

UDT user data structure (udtUser_t)

The data structure for one UDT user. Normally, an array of these data structures is created, with one element for each user. The UDT user data structure contains the following:

Type Name Description
char * userName a user-specified character string to identify this user or phrase
char * tag a tag field used by RTA (experimental)
unsigned short numEnroll the number of enrollment recordings that exist for this user. This value must be the same as the number of elements allocated for the 'enroll' array.
udtEnrollment_t * enroll data structure to hold each enrollment recording. It is an array of length 'numEnroll'.
char * reserved1 reserved field. Set to NULL.
char * reserved2 reserved field. Set to NULL.

where the udtEnrollment_t data structure is an array of 'numEnroll' elements, and contains the following (one for each enrollment recording):

The length of this array must be equal to the value of the 'audioLen' field.
Type Name Description
short * audio an array containing one enrollment recording in PCM format (with one short integer per sample). The sampling frequency must be the same that was specified in the call to thfUdtCreate.
unsigned long audioLen the number of samples in the 'audio' array. This value must be the number of short integers allocated for the 'audio' array field.
unsigned long audioSignature checksum used to keep audio and internal cache in sync. This field is automatically written when calling thfUdtCheckEnrollments.
char * filename an optional filename to save or read this enrollment recording. If not used, this value should be NULL.
char * reserved1 reserved field. Set to NULL.
char * pronun optional input field used to specify pronunciation to be used during enrollment. This pronunciation can be generated using a phoneme recognizer.
