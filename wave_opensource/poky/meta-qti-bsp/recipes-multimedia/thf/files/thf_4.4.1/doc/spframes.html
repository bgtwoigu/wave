How to Create a Sematic Parser Frames file

A 'frame' in the Semantic Parser is a set of related nets. It generally represents a basic type or action for the task. It is the top level of the parse, meaning that in order for a parse to be valid, the top net must be in a frame. All nets in the frame must also be defined in the grammar file. See the Phoenix Parser User Manual for more information on how to write the frame file.

The syntax for a frame definition is as follows: 

#comment
#include <external frame file name>

FRAME: <frame name>
NETS:
    [<net name>]
    [<net name>]
    [<net name>]
    ...
;

Lines beginning with a '#' are comments. Other frame files can be included with the '#include' keyword. These are prefixed with the same directory path given to thfSemParserCreateFromFile. The frame name is given after the FRAME: keyword. A list of the nets in this frame are given after the NETS: keyword. The frame definition must end with a semi-colon before beginning a new frame definition.
