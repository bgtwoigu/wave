How to Create an SDK Session

Before you can begin using the SDK, you must first create an SDK session. The SDK session encapsulates session-specific data and is represented by a session object (see thf_t). A handle to the session object is required as an input parameter to most SDK functions.

You can have multiple active sessions in your application. Each object created through the SDK is session specific, and results are unpredictable should they be used in unrelated sessions.

Your application code should begin with call to thfSessionCreate and end with a call to thfSessionDestroy.
