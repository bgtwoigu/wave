thfSpeakerInit

Syntax

#include <trulyhandsfree.h>
int thfSpeakerInit(thf_t **thf, recog_t *r, unsigned short sID, 
                   unsigned short wantNumRecordings);

Description

Creates a speaker-verification and speaker-ID object within the given recognizer object, and initializes internal data structures. This function must be called in the following circumstances: Before calling any other thfSpeaker function; After initializing a recognizer with thfRecogInit. Also, the recognizer must be initialized with the flag RECOG_KEEP_WAVE_WORD_PHONEME, as described in thfRecogInit. Finally, with a single user and the recommended API for SV, the phrasespot delay can be set as short as 15 msec (but 0 msec should not be used). With multiple users and/or the low-level API for SV, the phrasespot delay should be set to at least 250 msec to maintain accuracy. The delay can be configured using thfPhrasespotConfigSet.

The first speaker who will be enrolled is assigned an identification number equal to the 'sID' parameter. The 'wantNumRecordings' parameter specifies how many recordings of the target word or phrase are desired for enrollment. If more recordings are obtained than is specified in 'wantNumRecordings', then some recordings are discarded, as explained in thfSpeakerStoreLastRecording.

Parameters

Type Name Dir. Description
thf_t * *thf in Session object.
recog_t * r in Recognizer object.
unsigned short sID in The identification (integer value) assigned to the first speaker
unsigned short wantNumRecordings in How many recordings are to be collected for enrollment of each speaker

Return

Returns 1 on success or 0 on error.

Errors

String Description
recog object not found Invalid recognizer object handle.
thfRecogInit keepwave arg must be RECOG_KEEP_WAVE_WORD_PHONEME The recognizer was initialized with a flag other than RECOG_KEEP_WAVE_WORD_PHONEME.
internal error: search ROM struct not found The search object was found, but the ROM struct within this object was not found. This might indicate corrupt data.
