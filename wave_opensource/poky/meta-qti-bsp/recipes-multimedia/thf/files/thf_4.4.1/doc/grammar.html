Syntax of Grammar Specification

A grammar specification is a set of rules that describe the sequence of words that make up the recognition vocabulary. The rules typically employ variables to represent sub-sequences of words. The following syntactic conventions are used to specify grammars:
  • A grammar is a series of rules representing variable definitions, where the final rule represents the entire grammar.
  • A rule is of the form: 
    X = a b ... c;
    where X is the defined variable. Variable references must include the $ prefix (e.g., $X).
  • Each rule must terminate with a semicolon.
  • Variables must be defined before they can be referenced.
  • () Parentheses enclose items that are grouped together.
  • [] Square brackets enclose optional items.
  • <> Angle brackets denote one or more repetitions.
  • {} Curly braces denote zero or more repetitions (same as "[< >]").
  • | A pipe symbol separates alternative items.
  • %% A double percent suffix suppresses the word from being returned in the recognition result.
  • % A single percent sign conjoining two words forces the first word to be substituted for the second word in the recognition result (E.g., one%1 would recognize one but return 1).

The last two conventions are used to format the text returned by the recognition, and thus easing parsing in your application.

Typically the input utterance is a word or phrase that is framed by silence. Therefore, it is usual to include silence at the beginning and the end of a grammar.

Simple Grammar Specification:

The grammar comprises of a hierarchical set of phrase definitions, as follows: 
      size       = small | medium | large;
  topping    = vegetarian | pepperoni | cheese;
  pizzaorder = *sil%% ($size | $topping) [*sil%%] pizza *sil%%;

This grammar will allow 6 phrases, all ending in the word pizza, and starting with small, medium, large, vegetarian, pepperoni or cheese. Notice that the silence will not return anything in the result string if it matches because of the %% symbol.
